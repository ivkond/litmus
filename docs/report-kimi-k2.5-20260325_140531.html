Дашборд Grafana с метриками vLLM-сервера во время прогона. Характеризуют инфраструктуру, а не модель. Ключевые наблюдения: Finish Reason = length (модель упирается в max_tokens), TTFT P99 ~2s, 0 ошибок 5xx.

Выводы

  • 100% pass rate: Kimi-K2.5 прошла все 8 сценариев с обоими агентами (KiloCode и OpenCode). Все 80 из 80 баллов по критериям приёмки получены. Это сильный результат для бесплатной модели.
  • Самокоррекция: модель уверенно исправляет ошибки после провала тестов (self_correction 7/10). Типичный паттерн: неверный status code или формула → чтение assert-сообщения → точечное исправление.
  • Слабые стороны: избыточная многословность (verbosity 3/10), ошибки в первых попытках на API-задачах (status code 201 vs 200, extra field handling), слабая работа с импортами (4-5/10).
  • OpenCode быстрее на 33% (2:42 vs 3:36), но KiloCode чаще решает с первой попытки (75% vs 62.5%). Для задач, где retry дорогой, KiloCode предпочтительнее.
  • Инференс стабилен: 0 ошибок, TTFT P50 ~500ms. Finish Reason «length» указывает, что max_tokens стоит увеличить для снижения числа retry.
